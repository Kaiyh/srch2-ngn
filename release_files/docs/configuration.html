SRCH2 Configuration File Documentation

Version: 4.0, Date: September 15, 2013

Overview:

Welcome to Srch2 conf file documentation page. SRCH2 configuration file uses XML format. In this documentation XML tags which must be used in this file are introduced. The documentation starts by two major components of SRCH2 configuration file, config and schema. This brief introduction is followed by a detailed explanation of each one.

Major components: config vs. schema:

SRCH2 configuration file contains two top level XML tags; config and schema. Any tag which is to tune a part of engine is under 'config' while any tag which is used to describe data (or query) is placed under schema. For example, 'srch2home' tag which is used to specify the installation directory of SRCH2 engine is under 'config' while 'fields' which is used to declare data fields is under 'schema'.

Config:

In the following lines, each part explains each tag which can be provided under config.

SRCH2HOME:

SRCH2 Home directory
Required
This directory is going to be your srch2 home's directory. All other paths HAVE to be reletive to this path.
For Example:

To have "/usr/share/srch2/" as srch2 home directory, you should have the following line /usr/share/srch2/ All other paths in this conif file have to be relative to this path.
For Example:
If your data file is inside srch2 home directory, having the following absolute path "/usr/share/srch2/data/sample-data.json" In this config file it should be as "data/sample-data.json" 

    <srch2Home>/path/to/srch2/home/directory</srch2Home>

License File:

License Key File
Required
Email contact@srch2.com to get a key file.
Make sure to set the variable to the SRCH2 license key file.

    <licenseFile>/path/to/srch2_license_key.txt</licenseFile>

Listening Hostname and Port:

Hostname and Port Number
Required
Hostname and port number of the server. Make sure the port is open.

    <listeningHostname>127.0.0.1</listeningHostname>
    <listeningPort>8081</listeningPort>

Data Directory:

Index Directoy Path
Required
The file path to the folder where we store serialized index files. REQUIRED, when "index-load-or-create" is set to 1. 

    <dataDir>/path/to/data</dataDir>

Data File:

Data File
Required if "indexLoadCreate" is set to 0
Path for a data file to be loaded. The file must be in the json format. REQUIRED, when "index-load-or-create" is set to 0.

    <dataFile>/path/to/data.json</dataFile>

Field Boosts:

Boost Values
Optional

Boost values for searchable attributes. A boost value is in the range [1,100]. If the value is outside the range, it will be set to 100. Specify the boost values as comma separated entries.
For Example, the boosts for three searchable attributes can be specified as 

    <fieldBoost>relevance^10 name^7 title</fieldBoost>
  • fields are sparated by one space
  • if you do not provide any value for a field, it will be considered as 1 
    <fieldBoost></fieldBoost>
Record Boost Field:

Record Boost Field
Optional
Attribute used as the boost value of a record. Need to be float type in the json objects in the data file.

    <recordBoostField>relevance</recordBoostField>

Response Content:

Response Content
Optional

Format of each json search response.
You should set the type by one of the following options:

0: format "rid,edit_dist,matching_prefix,in_memory_record";
1: format "rid,edit_dist,matching_prefix".
2: format "rid,edit_dist,matching_prefix,feilds_to_return".
Defaults to 0
It is REQUIRED to set fields_to_return if search-response-format=2
These are attributes returned to the user.
For Example: 

    <responseContent type="2">lat,lng</responseContent>

or

    <responseContent type="0"></responseContent>

Fields:

This tag contains all 'field' tags.

    <fields>
    ...
    </fields>

Fields
Required
Fields we want to search on in addition to the primary key.

"name" : is the name of the field
"type" : is the type of the field. Supported types are:

  • Text
  • location_longtitude (For Geo search)
  • location_latitude (For Geo search)
  • integer
  • float
  • time

"indexed" : If you want to search on a field, indexed should be true. A field can only be used for sort, filtering and facet if its 'indexed' property is false. If you need a field to be used in keyword search AND also for sort,filter and facet, you should include two instances of the field one with indexed="true" (for keyword search) and one with indexed="false" (for sort and facet).
"default" : The default value which is used if this field is empty.
"required": The flag which indicates whether this field can be empty or not. It is false by default.

Note:

  • If you want to have geo search, you should have two fields with the types of ocation_latitude and location_longtitude. Make sure to set both latitude and longitude.
  • The values of these geo fields should be float in the JSON (input) file.
  • Primary key is optional to be included here. It can be included here to set its indexed property to true.
  • Type is always text for primary key, it is required and there is no need for default value. If primary key is included here and indexed="false" it will be also used for sort,filtering and facet.
    <field name="category" type="text" indexed="true" />
    <field name="name" type="text" indexed="true" />
    <field name="model" default="benz" required="true" type="text" indexed="false"/>
    <field name="price" default="200" required="true" type="float" indexed="false"/>
    <field name="likes" default="0" required="true" type="integer" indexed="false"/>

Primary Key Required

Primary key attribute of the data

    <uniqueKey>id</uniqueKey>

Enable Facet:

Facet enabled flag
Optional
Indicates whether faceted search is allowed or not.

Note:
- values are false (disabled) and true (enabled)
- By default it is disabled.

    <facetEnabled>true</facetEnabled>start

Facet Fields:

This tag contains all 'facetField' tags.

    <facetFields>
    ...
    </facetFields>

Facet fields
Required if facet is enabled.

name: The name of the field which is used for this facet.
facetType: Facet type which is Categorical or Range. Enter "cateorical" or "range"
facetStart: The start point of making range facets. REQUIRED for range facets. The value will be ignored for categorical facets.
facetEnd: The end point of making range facets. REQUIRED for range facets. The value will be ignored for categorical facets.
facetGap: The gap used for making range facets. REQUIRED for range facets. The value will be ignored for categorical facets.

Note:
- All facetFields must be declared in fields tag as well (with indexed="false") 

    <facetField name="likes" facetType="range" facetStart="0" facetEnd="50" facetGap="10" />
    <facetField name="model" facetType="categorical" />
    <facetField name="price" facetType="range" facetStart="0" facetEnd="100" facetGap="10" />

Synonym Filter:

Synonym Filter Optional

Enable/Disabling the Synonym Filter By providing "synonym" file.
For Example:
Disable Synonym Filter: 

    <filter name="SynonymFilter" synonyms=""  expand="true"/>

To keep the original word in this filter, set the "expand" attribute as "True" Otherwise set it as "False". 

    <filter name="SynonymFilter" synonyms=""  expand="true"/>
